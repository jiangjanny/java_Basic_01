## SQL 
### SQL 通用語法
1. SQL 語句可以單行或多行書寫，以分號結尾
2. SQL 語句可以使用空格 / 縮進還增加語句的可讀性
3. MySQL 數據庫的SQL 語句不區分大小寫，關鍵字建議使用大寫
   - 單行註解: -- 註解內容 或 # 註解內容 (MySQL 特有)
   - 多行註解: /* 註解內容 */
  
### SQL 分類
1. DDL (Data Definition Language) : 數據定義語言，用來定義數據庫對象(數據庫 / 表 / 字段) <img width="532" height="117" alt="image" src="https://github.com/user-attachments/assets/2bbfd858-6349-447f-a840-e55d2698f5ea" />

2. DML (Data Manipulation Language) : 數據庫操作語言，用來對數據庫表中的數據進行曾刪改
3. DQL (Data Query Language) : 數據查詢語言，用來查詢數據庫中表的紀錄
4. DCL (Data Control Language) : 數據控制語言，用來創建數據庫用戶，控制數據庫的訪問權限

#### DDL - 數據庫操作
1. 查詢 
   - 查詢所有數據庫  : SHOW DATABASES;
   - 查詢當前數據庫 : SELECT DATABASE();
2. 創建
   - CREATE DATABASE [IF NOT EXISTS] 數據庫名 [DEFAULT CHARSET 字符集] [COLLATE 排序規則];
3. 刪除
   - DROP DATABASE [IF EXISTS] 數據庫名;
4. 使用
   - USE 數據庫名;
### DDL - 表操作- 查詢
1. 查詢當前數據庫所有表 : SHOW TABLES;
2.  查詢表結構 : DESC 表明;
3.  查詢指定表的建表語句 : SHOW CREATE TABLE 表名;

### DDL - 表操作-創建
在 SQL 的 **DDL（Data Definition Language）** 中，`CREATE TABLE` 用於建立新資料表。以下是完整語法與範例說明。

---

#### 🔹 基本語法

```sql
CREATE TABLE 表名 (
    欄位1 資料型別 [限制條件],
    欄位2 資料型別 [限制條件],
    ...
);
```

---

#### 🔹 常見資料型別

| 型別           | 說明     |
| ------------ | ------ |
| INT          | 整數     |
| VARCHAR(n)   | 可變長度字串 |
| CHAR(n)      | 固定長度字串 |
| DATE         | 日期     |
| DATETIME     | 日期時間   |
| DECIMAL(m,n) | 精確數值   |
| BOOLEAN      | 布林值    |

---

#### 🔹 常見限制條件（Constraints）

| 限制             | 說明          |
| -------------- | ----------- |
| PRIMARY KEY    | 主鍵          |
| NOT NULL       | 不可為空        |
| UNIQUE         | 唯一值         |
| DEFAULT        | 預設值         |
| AUTO_INCREMENT | 自動遞增（MySQL） |
| FOREIGN KEY    | 外鍵          |

---

#### 🔹 範例 1：建立基本資料表

```sql
CREATE TABLE students (
    id INT PRIMARY KEY,
    name VARCHAR(50) NOT NULL,
    birthday DATE,
    email VARCHAR(100) UNIQUE
);
```

---

#### 🔹 範例 2：含 AUTO_INCREMENT（MySQL）

```sql
CREATE TABLE users (
    user_id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    password VARCHAR(100) NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

---

##### 🔹 範例 3：含外鍵

```sql
CREATE TABLE orders (
    order_id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT,
    order_date DATE,
    FOREIGN KEY (user_id) REFERENCES users(user_id)
);
```

---

#### 🔹 如果資料表已存在避免錯誤

```sql
CREATE TABLE IF NOT EXISTS table_name (
    ...
);
```

### DDL - 數據類型
在 **MySQL** 中，資料型別（Data Types）主要分為以下幾大類：

---

#### 🔹 一、數值型別（Numeric Types）

| 型別            | 儲存空間    | 範圍（有符號）          |
| ------------- | ------- | ---------------- |

### DDL - 表操作 - 修改
添加字段
ALTER TABLE 表名 ADD 欄位名稱 資料型別;

在資料庫中，`ALTER TABLE` 是用來**修改已存在資料表結構**的 DDL（Data Definition Language）指令。

---

#### 🔹 一、基本語法：新增欄位（添加字段）

```sql
ALTER TABLE 表名
ADD 欄位名稱 資料型別;
```

---

#### 🔹 二、範例說明

假設原本有一個 `students` 表：

```sql
CREATE TABLE students (
    id INT PRIMARY KEY,
    name VARCHAR(50)
);
```

##### ✅ 1️⃣ 新增單一欄位

```sql
ALTER TABLE students
ADD age INT;
```

👉 在 `students` 表中新增 `age` 欄位。

---

##### ✅ 2️⃣ 新增多個欄位（MySQL 寫法）

```sql
ALTER TABLE students
ADD email VARCHAR(100),
ADD phone VARCHAR(20);
```

---

##### ✅ 3️⃣ 新增欄位並設定預設值

```sql
ALTER TABLE students
ADD status VARCHAR(20) DEFAULT 'active';
```

---

##### ✅ 4️⃣ 新增欄位並設定 NOT NULL

⚠️ 注意：若表內已有資料，新增 NOT NULL 欄位必須給預設值。

```sql
ALTER TABLE students
ADD gender CHAR(1) NOT NULL DEFAULT 'M';
```

---

#### 🔹 三、不同資料庫差異

| 資料庫        | 新增多欄位語法           |
| ---------- | ----------------- |
| MySQL      | 使用多個 `ADD`        |
| PostgreSQL | 使用多個 `ADD COLUMN` |
| SQL Server | 可同時新增多欄位          |
| Oracle     | 使用括號新增多欄位         |

### DML  介紹
> DML (Data Manipulation Language 數據操作語言), 用來對數據庫中表的數據紀錄進行增刪操作
> 
| 操作   | SQL 關鍵字 | 作用     |
| ---- | ------- | ------ |
| 添加數據 | INSERT  | 插入新資料  |
| 修改數據 | UPDATE  | 更新現有資料 |
| 刪除數據 | DELETE  | 刪除資料   |

---

#### 1️⃣ 添加數據：`INSERT`

##### 🔹 基本語法

```sql
INSERT INTO 表名 (欄位1, 欄位2, ...)
VALUES (值1, 值2, ...);
```

###### 🔹 範例

假設有表 `tb_user`：

```sql
CREATE TABLE tb_user (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    age INT
);
```

新增一筆資料：

```sql
INSERT INTO tb_user (id, name, age)
VALUES (1, 'Alice', 25);
```

新增多筆資料：

```sql
INSERT INTO tb_user (id, name, age)
VALUES 
(2, 'Bob', 30),
(3, 'Charlie', 22);
```

---

#### 2️⃣ 修改數據：`UPDATE`

##### 🔹 基本語法

```sql
UPDATE 表名
SET 欄位1 = 值1, 欄位2 = 值2, ...
WHERE 條件;
```

> ⚠️ **重要**：沒有 `WHERE` 條件會更新整張表！

###### 🔹 範例

將 `Alice` 的年齡修改為 26：

```sql
UPDATE tb_user
SET age = 26
WHERE name = 'Alice';
```

將所有使用者年齡加 1：

```sql
UPDATE tb_user
SET age = age + 1;
```

---

#### 3️⃣ 刪除數據：`DELETE`

##### 🔹 基本語法

```sql
DELETE FROM 表名
WHERE 條件;
```

> ⚠️ **重要**：沒有 `WHERE` 條件會刪除整張表的資料！

###### 🔹 範例

刪除名字是 `Bob` 的使用者：

```sql
DELETE FROM tb_user
WHERE name = 'Bob';
LIMIT 1;
```

刪除整張表所有資料：

```sql
DELETE FROM tb_user;
```

#### LIMIT 是「數量限制」，Safe Update 會允許加上 LIMIT 的 DELETE 或 UPDATE。

> 安全刪除建議：最好用主鍵或索引精確刪除或加 LIMIT 逐筆刪除先 SELECT 確認要刪的資料

## 1️⃣ 基礎查詢 (Basic SELECT)

查詢整個表或部分欄位：

```sql
-- 查詢整個表
SELECT * FROM users;

-- 查詢指定欄位
SELECT id, name, email FROM users;
```

---

## 2️⃣ 條件篩選 (WHERE / 条件過濾)

用 `WHERE` 來篩選符合條件的資料：

```sql
-- 篩選單一條件
SELECT * FROM users
WHERE age > 18;

-- 多條件 AND / OR
SELECT * FROM users
WHERE age > 18 AND status = 'active';

-- 模糊搜尋 LIKE
SELECT * FROM users
WHERE name LIKE 'A%';  -- 以 A 開頭
```

---

## 3️⃣ 聚合函數 (Aggregate Functions)

計算統計值，常用聚合函數包括：`COUNT()`, `SUM()`, `AVG()`, `MAX()`, `MIN()`。

```sql
-- 計算總數
SELECT COUNT(*) AS total_users FROM users;

-- 計算平均年齡
SELECT AVG(age) AS avg_age FROM users;

-- 計算總銷售額
SELECT SUM(sales) AS total_sales FROM orders;
```

---

## 4️⃣ 分組 (GROUP BY / HAVING)

當要依某個欄位分組並計算聚合值：

```sql
-- 按部門計算人數
SELECT department, COUNT(*) AS employee_count
FROM employees
GROUP BY department;

-- 分組後篩選 (HAVING)
SELECT department, AVG(salary) AS avg_salary
FROM employees
GROUP BY department
HAVING AVG(salary) > 50000;
```

> 注意：`HAVING` 是對分組結果篩選，`WHERE` 是對原始資料篩選。

---

## 5️⃣ 排序 (ORDER BY)

按照某個欄位排序，預設升序 `ASC`，降序 `DESC`：

```sql
-- 按年齡升序
SELECT * FROM users
ORDER BY age ASC;

-- 按薪水降序
SELECT * FROM employees
ORDER BY salary DESC;
```

---

## 6️⃣ 分頁 (LIMIT / OFFSET)

在查詢大量資料時，常用分頁：

```sql
-- 取前 10 筆
SELECT * FROM users
LIMIT 10;

-- 跳過前 10 筆，取 10 筆
SELECT * FROM users
LIMIT 10 OFFSET 10;
```

> MySQL 也可寫成 `LIMIT 10, 10`，表示從第 11 筆開始取 10 筆。

### DCL（Data Control Language，資料控制語言）
好的，我們來整理 MySQL 的 **DCL（Data Control Language，資料控制語言）**。DCL 主要用於 **權限管理和安全控制**，不像 DQL 是查資料，DCL 是管誰可以做什麼。

---

#### 1️⃣ 基本概念

DCL 主要有兩個指令：

| 指令       | 功能      |
| -------- | ------- |
| `GRANT`  | 給予使用者權限 |
| `REVOKE` | 收回使用者權限 |

---

#### 2️⃣ GRANT — 給予權限

```sql
-- 給使用者某個權限
GRANT SELECT, INSERT ON database_name.* TO 'username'@'host';

-- 給予所有權限
GRANT ALL PRIVILEGES ON database_name.* TO 'username'@'host';

-- 立即生效
FLUSH PRIVILEGES;
```

說明：

* `'username'@'host'`：MySQL 帳號 + 可連線位置

  * 例如：`'tom'@'localhost'` 或 `'tom'@'%'` (`%` 表示任何位置)
* `database_name.*`：指定整個資料庫
* 權限類型常用：

  * `SELECT`：查詢
  * `INSERT`：新增
  * `UPDATE`：更新
  * `DELETE`：刪除
  * `ALL PRIVILEGES`：全部權限

---

#### 3️⃣ REVOKE — 收回權限

```sql
-- 收回使用者的 SELECT 權限
REVOKE SELECT ON database_name.* FROM 'username'@'host';

-- 收回所有權限
REVOKE ALL PRIVILEGES ON database_name.* FROM 'username'@'host';

-- 立即生效
FLUSH PRIVILEGES;
```

> 注意：`REVOKE` 收回後，也可以透過 `GRANT` 重新給予。

---

#### 4️⃣ 查看使用者權限

```sql
-- 查看當前使用者權限
SHOW GRANTS;

-- 查看指定使用者權限
SHOW GRANTS FOR 'username'@'host';
```
